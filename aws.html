AWS Video 2: Cloud Computing & Pricing ‚ñ∂

‚òÅÔ∏è What is Cloud Computing?

Cloud computing is the delivery of computing services‚Äîservers, storage, databases, networking, software‚Äîover the Internet ("the cloud") to offer faster innovation, flexible resources, and economies of scale.

üì¶ Types of Services

  • IaaS (Infrastructure as a Service): Basic infrastructure like VMs, storage, networks (e.g., EC2).
  • PaaS (Platform as a Service): Tools and services to deploy applications (e.g., Elastic Beanstalk).
  • SaaS (Software as a Service): Ready-to-use software delivered over the internet (e.g., Gmail).

üè† Deployment Models

  • Private Cloud: Cloud resources used exclusively by one business.
  • Public Cloud: Services delivered over the public internet (AWS, Azure).
  • Hybrid Cloud: Mix of private and public cloud resources.

üí≤ AWS Pricing

  • Pay-as-you-go model ‚Äì only pay for what you use.
  • Free Tier available for new users.
  • Pricing calculators and savings plans help manage costs.

üåç AWS Regions & Availability Zones

AWS divides the globe into regions (like us-east-1, ap-south-1). Each region contains multiple isolated Availability Zones (AZs) to ensure high availability, redundancy, and fault tolerance.

AWS Video 3: EC2 Introduction & Hands-On ‚ñ∂

üñ•Ô∏è What is EC2?

Amazon EC2 (Elastic Compute Cloud) is a web service that provides secure, resizable compute capacity in the cloud. It is designed to make web-scale cloud computing easier for developers.

üîç Overview

  • Launch virtual servers (instances) in minutes.
  • Choose from a variety of OS: Linux, Windows, etc.
  • Customize CPU, memory, storage, and networking.
  • Pay only for what you use (Free Tier available).

üåü Advantages

  • Highly scalable and flexible infrastructure.
  • Secure with options for firewalls, key pairs, and IAM.
  • Elastic ‚Äì Start, stop, resize instances anytime.
  • Used by companies like Netflix, NASA, Airbnb, BMW.
  • Integrated with other AWS services like S3, RDS, etc.

‚öôÔ∏è Steps to Launch EC2 Instance

  1. Log in to your AWS account (or create a Free Tier account).
  2. Navigate to EC2 service from the AWS Management Console.
  3. Click "Launch Instance".
  4. Select an Amazon Machine Image (AMI) ‚Äì e.g., Amazon Linux.
  5. Choose an instance type ‚Äì e.g., t2.micro (Free Tier eligible).
  6. Configure instance details and network settings.
  7. Add storage (optional ‚Äì default 8 GB).
  8. Tag your instance (e.g., Name = MyEC2).
  9. Configure Security Group ‚Äì allow SSH (port 22).
  10. Launch instance and download Key Pair (.pem file).
  11. Connect to your instance via SSH using the .pem file.

Now your cloud-based server is ready to use! üéâ

AWS Video 5: Security Groups & Hosting a Website on EC2 ‚ñ∂

üõ°Ô∏è What is a Security Group in AWS?

A Security Group (SG) is a stateful virtual firewall attached to your EC2 instance‚Äôs network interface. It controls which traffic is allowed in (inbound) and allowed out (outbound). By default, all inbound is denied and all outbound is allowed (you can change this).

üîÅ Inbound & Outbound Rules Explained

  • Inbound rules let specific traffic reach your instance. Examples:
    • SSH: TCP 22 from My IP (safer) to administer Linux.
    • HTTP: TCP 80 from 0.0.0.0/0 (public web access).
    • HTTPS: TCP 443 from 0.0.0.0/0 for secure web.
  • Outbound rules define what your instance can reach (e.g., the internet). Most apps keep ‚ÄúAll traffic‚Äù allowed.
  • Stateful means return traffic is automatically allowed‚Äîno need to add reverse rules.

üß± How to Create a Custom Security Group

  1. AWS Console ‚Üí EC2 ‚Üí Security Groups ‚Üí Create security group.
  2. Name it (e.g., web-sg), select the correct VPC.
  3. Add Inbound rules:
    • HTTP, TCP 80, Source: 0.0.0.0/0
    • HTTPS, TCP 443, Source: 0.0.0.0/0 (optional)
    • SSH, TCP 22, Source: My IP (recommended)
  4. Leave Outbound as ‚ÄúAll traffic‚Äù (or restrict if required).
  5. Click Create security group.

üîó Assigning the Security Group to an EC2 Instance

  • During launch: On the ‚ÄúSecurity group‚Äù step, choose Select existing and pick web-sg.
  • After launch: EC2 ‚Üí Instances ‚Üí select instance ‚Üí Actions ‚Üí Security ‚Üí Change security groups ‚Üí add web-sg ‚Üí Save.

üåê Installing & Running a Sample Web Server

SSH into the instance, then use one of the quick setups below.

Option A ‚Äî Apache on Ubuntu/Debian

sudo apt update
sudo apt install -y apache2
sudo systemctl enable --now apache2
echo "<h1>Hello from Apache on EC2</h1>" | sudo tee /var/www/html/index.html

Option B ‚Äî Apache on Amazon Linux 2

sudo yum update -y
sudo yum install -y httpd
sudo systemctl enable --now httpd
echo "<h1>Hello from Apache on Amazon Linux 2</h1>" | sudo tee /var/www/html/index.html

Option C ‚Äî Simple Node.js Server (port 3000)

curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo -E bash -   # (Amazon Linux/RHEL) use apt setup on Ubuntu
sudo yum install -y nodejs || sudo apt install -y nodejs npm
cat > server.js <<'EOF'
const http = require('http');
const server = http.createServer((req, res) => {
  res.end('Hello from Node.js on EC2');
});
server.listen(3000, '0.0.0.0');
console.log('Server on port 3000');
EOF
node server.js

Note: If you use Node on port 3000, add an inbound rule for TCP 3000 or place Node behind Apache/Nginx as a reverse proxy.

üîé Accessing Your Hosted Website via Public IP

  1. In EC2 ‚Üí Instances, copy the instance‚Äôs Public IPv4 address.
  2. Ensure your SG allows the port you‚Äôre using:
    • Apache: TCP 80 (and 443 if HTTPS).
    • Node example: TCP 3000.
  3. Open a browser:
    • http://<PUBLIC_IP>/ (Apache)
    • http://<PUBLIC_IP>:3000/ (Node)
  4. If it doesn‚Äôt load, re-check SG inbound rules, instance running state, and that the service is active.
