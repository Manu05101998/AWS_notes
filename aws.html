Cloud computing is the delivery of computing services‚Äîservers, storage, databases, networking, software‚Äîover the Internet ("the cloud") to offer faster innovation, flexible resources, and economies of scale.

üì¶ Types of Services

  • IaaS (Infrastructure as a Service): Basic infrastructure like VMs, storage, networks (e.g., EC2).
  • PaaS (Platform as a Service): Tools and services to deploy applications (e.g., Elastic Beanstalk).
  • SaaS (Software as a Service): Ready-to-use software delivered over the internet (e.g., Gmail).

üè† Deployment Models

  • Private Cloud: Cloud resources used exclusively by one business.
  • Public Cloud: Services delivered over the public internet (AWS, Azure).
  • Hybrid Cloud: Mix of private and public cloud resources.

üí≤ AWS Pricing

  • Pay-as-you-go model ‚Äì only pay for what you use.
  • Free Tier available for new users.
  • Pricing calculators and savings plans help manage costs.

üåç AWS Regions & Availability Zones

AWS divides the globe into regions (like us-east-1, ap-south-1). Each region contains multiple isolated Availability Zones (AZs) to ensure high availability, redundancy, and fault tolerance.

üñ•Ô∏è What is EC2?

Amazon EC2 (Elastic Compute Cloud) is a web service that provides secure, resizable compute capacity in the cloud. It is designed to make web-scale cloud computing easier for developers.

üîç Overview

  • Launch virtual servers (instances) in minutes.
  • Choose from a variety of OS: Linux, Windows, etc.
  • Customize CPU, memory, storage, and networking.
  • Pay only for what you use (Free Tier available).

üåü Advantages

  • Highly scalable and flexible infrastructure.
  • Secure with options for firewalls, key pairs, and IAM.
  • Elastic ‚Äì Start, stop, resize instances anytime.
  • Used by companies like Netflix, NASA, Airbnb, BMW.
  • Integrated with other AWS services like S3, RDS, etc.

‚öôÔ∏è Steps to Launch EC2 Instance

  1. Log in to your AWS account (or create a Free Tier account).
  2. Navigate to EC2 service from the AWS Management Console.
  3. Click "Launch Instance".
  4. Select an Amazon Machine Image (AMI) ‚Äì e.g., Amazon Linux.
  5. Choose an instance type ‚Äì e.g., t2.micro (Free Tier eligible).
  6. Configure instance details and network settings.
  7. Add storage (optional ‚Äì default 8 GB).
  8. Tag your instance (e.g., Name = MyEC2).
  9. Configure Security Group ‚Äì allow SSH (port 22).
  10. Launch instance and download Key Pair (.pem file).
  11. Connect to your instance via SSH using the .pem file.
